# 全局代码规范（所有项目默认遵守）

## 1. 命名规范
- 变量/函数：**camelCase**
- 类/结构体：**PascalCase**
- 常量：**UPPER_SNAKE_CASE**
- 文件名：**kebab-case 或 snake_case**
- 禁止拼音、缩写不明确、无意义命名（如 a, b, tmp）

## 2. 注释规范
- 所有**公共方法/类/接口**必须有完整注释（功能、参数、返回值、异常、示例）
- 复杂逻辑/算法必须加**行内注释**说明意图
- 注释要说明**为什么**，而不是**做什么**
- 禁止无用注释、掉书袋、复制代码当注释

## 3. 代码结构
- 单一职责：一个函数/类只做一件事
- 函数长度：**≤ 50 行**，超过必须拆分
- 嵌套层级：**≤ 3 层**（if/for/try）
- 参数个数：**≤ 5 个**，多用对象/结构体传参

## 4. 错误处理
- 禁止空 catch / 空 try
- 所有外部调用（网络/文件/DB）必须**捕获异常 + 日志 + 友好返回**
- 异常要**精准捕获**，不要 catch-all
- 返回错误码/信息要**统一、可读、可排查**

## 5. 日志规范
- 关键节点（入口、出口、异常、参数）必须打日志
- 日志级别：debug/info/warn/error 正确使用
- 日志内容：时间、模块、函数、参数、结果、异常栈
- 禁止日志打印敏感信息（密码、密钥、手机号）

## 6. 安全规范
- 输入必须**校验、过滤、转义**
- SQL/NoSQL 必须用**参数化查询**，禁止拼接
- 接口必须**鉴权、防重放、防越权**
- 禁止硬编码密钥、密码、token
- 依赖包定期扫描漏洞

## 7. 性能规范
- 避免循环内查询/IO
- 大列表必须**分页/流式处理**
- 字符串拼接用高效方式（如 join）
- 禁止内存泄漏（未释放句柄、全局缓存无限增长）

## 8. 通用约束
- 代码必须**可格式化、可lint**
- 提交前必须**自审：无warning、无console.log、无TODO**
- 优先用**标准库/成熟组件**，不重复造轮子
- 兼容主流版本，不写死环境
