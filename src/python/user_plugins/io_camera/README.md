# 图像相机节点包使用指南

## 📦 节点列表

### 1. 加载图像 (ImageLoadNode)
- **功能**: 从本地文件加载图像
- **输入**: 无
- **输出**: 输出图像
- **参数**: 
  - `file_path`: 图像文件路径

**使用方法**:
1. 拖拽"加载图像"节点到画布
2. 在属性面板中输入图像文件路径，或**双击节点**选择文件
3. 运行工作流

---

### 2. 保存图像 (ImageSaveNode)
- **功能**: 将图像保存到本地文件
- **输入**: 输入图像
- **输出**: 状态信息
- **参数**: 
  - `save_path`: 保存路径
  - `status`: 状态显示（只读）

**使用方法**:
1. 拖拽"保存图像"节点到画布
2. 连接上游节点的输出
3. 在属性面板中输入保存路径，或**双击节点**选择保存位置
4. 运行工作流

---

### 3. 图像显示 (ImageViewNode) ⭐ 新增
- **功能**: 显示图像并支持双击预览
- **输入**: 输入图像
- **输出**: 输出图像（可继续传递给下游节点）
- **参数**: 
  - `status`: 显示图像尺寸、通道数、数据类型等信息

**使用方法**:
1. 拖拽"图像显示"节点到画布
2. 连接上游节点的输出到"输入图像"端口
3. 运行工作流
4. **双击节点**打开图像预览窗口

**预览窗口功能**:
- ✅ 缩放控制：滚轮/Ctrl+滚轮/+/-键
- ✅ 平移：空格+鼠标拖拽
- ✅ 刷新：点击"🔄 刷新预览"按钮获取最新图像
- ✅ 保存：点击"💾 保存图像"按钮
- ✅ 多窗口：可同时打开多个预览窗口
- ✅ 智能管理：重复双击同一节点会刷新已有窗口而非创建新窗口

---

## 🔧 双击交互功能说明

### 支持的节点类型及双击行为

| 节点类型 | 双击行为 | 说明 |
|---------|---------|------|
| ImageLoadNode | 弹出文件打开对话框 | 快速选择要加载的图像文件 |
| ImageSaveNode | 弹出文件保存对话框 | 快速选择保存路径 |
| ImageViewNode | 打开图像预览窗口 | 查看处理后的图像结果 |

### 技术实现

主窗口通过 `_on_node_double_clicked()` 方法统一处理所有节点的双击事件：

```python
def _on_node_double_clicked(self, node, workflow=None):
    # 1. 获取节点类型标识
    node_type = node.type_()
    
    # 2. 根据节点类型执行不同操作
    if "ImageLoadNode" in node_type:
        self._on_browse_image_file(node)  # 打开文件选择对话框
    elif "ImageSaveNode" in node_type:
        self._on_select_save_path(node)   # 打开保存路径对话框
    elif "ImageViewNode" in node_type:
        # 打开图像预览窗口
        image = node.get_cached_image()
        if image is not None:
            dialog = ImagePreviewDialog(image, node=node, ...)
            dialog.show()
```

## 💡 使用技巧

### 1. 快速构建图像处理流程

```
[加载图像] → [灰度化] → [Canny边缘检测] → [图像显示]
     ↓                                          ↓
  双击选择文件                              双击查看结果
```

### 2. 实时预览工作流程

1. 添加"图像显示"节点到关键处理步骤后
2. 运行工作流
3. 双击任意"图像显示"节点查看中间结果
4. 修改上游节点参数后重新运行
5. 点击预览窗口的"🔄 刷新预览"更新图像

### 3. 多阶段对比

可以同时打开多个"图像显示"节点的预览窗口，对比不同处理阶段的效果：

```

   ↓              ↓             ↓             ↓
 [显示1]      [显示2]       [显示3]       [显示4]
   ↓              ↓             ↓             ↓
 双击预览      双击预览      双击预览      双击预览
```

## ❓ 常见问题

### Q1: 双击节点没有反应？
**A**: 请检查：
1. 节点是否已正确连接到上游节点
2. 是否已运行工作流（节点需要处理过图像才有缓存）
3. 查看控制台输出的调试信息

### Q2: 预览窗口显示"该节点尚未处理图像数据"？
**A**: 需要先运行工作流，让节点处理图像并缓存结果。

### Q3: 如何关闭预览窗口？
**A**: 点击窗口右上角的关闭按钮，或按 Alt+F4。

### Q4: 可以同时打开多少个预览窗口？
**A**: 理论上无限制，但建议同时打开不超过5个以避免内存占用过高。

### Q5: 预览窗口中的图像不清晰？
**A**: 使用缩放功能：
- 点击"📏 原始大小"查看100%尺寸
- 使用滚轮或+/-键调整缩放比例
- 点击"⊞ 适应窗口"自动调整

## 🎯 最佳实践

1. **在每个关键处理步骤后添加"图像显示"节点**，便于调试和验证
2. **使用有意义的节点名称**，方便识别不同的预览窗口
3. **定期清理不再需要的预览窗口**，释放内存
4. **利用刷新功能**实时查看参数调整效果
5. **结合属性面板的状态信息**了解图像详细参数
