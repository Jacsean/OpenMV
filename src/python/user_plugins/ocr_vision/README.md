# OCR 文字识别插件 (ocr_vision)

基于 **PaddleOCR** 实现的高精度中文/英文文字识别插件，支持通用文字识别、表格识别和版面分析。

## 🎯 功能特性

### ✅ 核心能力
- **高精度识别**: 基于 PaddleOCR，中文识别准确率 >95%
- **多语言支持**: 支持 80+ 语言（中英文混合、日文、韩文等）
- **表格识别**: 自动检测表格结构，导出 Excel/CSV/JSON
- **版面分析**: 智能识别文档结构（标题、段落、图片、表格）
- **GPU 加速**: 支持 CUDA 加速推理（可选）

### 🚀 技术优势
- **轻量级部署**: 推理节点仅需 CPU 2核 + 4GB 内存
- **模块化设计**: 3个独立节点，可按需组合
- **结构化输出**: JSON 格式结果，易于后续处理
- **可视化标注**: 自动生成带标注框的结果图像

## 🔧 安装依赖

### 方法 1：仅推理模式（推荐工厂部署）

```bash
pip install paddlepaddle>=2.4.0
pip install paddleocr>=2.6.0
```

**预计大小**: ~500MB  
**下载时间**: 约 10 分钟（宽带）

### 方法 2：完整套件（含 GPU 支持）

```bash
pip install paddlepaddle-gpu>=2.4.0
pip install paddleocr>=2.6.0
pip install openpyxl pandas
```

**预计大小**: ~2GB  
**下载时间**: 约 30 分钟（宽带）

### 验证安装

```python
from paddleocr import PaddleOCR
ocr = PaddleOCR(use_angle_cls=True, lang='ch')
print("✅ PaddleOCR 安装成功！")
```

## 🚀 快速开始

### 步骤 1：加载插件

1. 启动应用程序
2. 在节点库中找到 **"AI"** 标签页
3. 拖拽 **"OCR 文字识别"** 节点到画布

### 步骤 2：配置参数

双击节点打开属性面板：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| 语言类型 | 识别语言 | ch（中文）/ en（英文） |
| 检测阈值 | 文本检测灵敏度 | 0.3（平衡）/ 0.5（高精度） |
| 识别批次大小 | 批量处理数量 | 6（默认） |
| 使用 GPU 加速 | 启用 CUDA | False（CPU）/ True（GPU） |
| 显示可视化结果 | 绘制检测框 | True |

### 步骤 3：连接工作流

```
[输入图像] → [OCR 文字识别] → [识别文本] + [检测框坐标]
```

### 步骤 4：运行识别

1. 连接输入图像节点
2. 点击 **执行** 按钮
3. 查看输出结果

## 💡 使用示例

### 示例 1：通用文字识别

**场景**: 识别产品包装上的文字信息

```json
{
  "lang": "ch",
  "det_db_thresh": "0.3",
  "use_gpu": false,
  "show_visualization": true
}
```

**输出**:
- 识别文本: "产品名称: XXX\n生产日期: 2026-04-26"
- 检测框坐标: JSON 格式的边界框信息

### 示例 2：表格识别

**场景**: 数字化纸质报表

```json
{
  "export_format": "excel",
  "output_dir": "./ocr_tables",
  "use_gpu": false
}
```

**输出**:
- Excel 文件: `./ocr_tables/table_1.xlsx`
- JSON 数据: 表格结构和单元格信息

### 示例 3：版面分析

**场景**: 解析技术文档结构

```json
{
  "analysis_mode": "full",
  "extract_text": true,
  "show_visualization": true
}
```

**输出**:
- 布局 JSON: 包含标题、段落、图片的位置和层级
- 可视化图像: 彩色标注不同区域类型

## ⚡ 性能优化

### CPU 优化

| 优化项 | 建议 | 效果 |
|--------|------|------|
| 图像尺寸 | 缩小至 1000-1500px | 速度提升 30-50% |
| 检测阈值 | 提高至 0.5 | 减少误检，提升速度 |
| 批次大小 | 降低至 4 | 减少内存占用 |

### GPU 优化

| 优化项 | 建议 | 效果 |
|--------|------|------|
| 模型选择 | 使用超轻量模型 | 速度提升 2-3x |
| 批处理 | 增大批次至 16 | 吞吐量提升 50% |
| 显存管理 | 关闭其他 GPU 应用 | 避免 OOM |

### 内存优化

- **懒加载**: 首次运行时下载模型（~100MB）
- **缓存机制**: 重复图像可缓存结果
- **及时释放**: 处理完成后清理临时数据

## ❓ 常见问题

### Q1: 首次运行很慢怎么办？

**A**: 首次运行会自动下载模型文件（约 100MB），请耐心等待。后续运行会快很多。

### Q2: 中文识别不准确？

**A**: 
1. 确保 `lang` 参数设置为 `"ch"`
2. 检查图像清晰度（建议分辨率 >1000px）
3. 调整 `det_db_thresh` 至 0.2-0.4
4. 尝试启用 GPU 加速

### Q3: 表格识别失败？

**A**: 
1. 确保图像中表格清晰可见
2. 避免倾斜或变形的表格
3. 尝试先使用通用 OCR 节点验证图像质量
4. 检查是否安装了 `openpyxl` 和 `pandas`

### Q4: 显存不足（OOM）？

**A**: 
1. 减小图像尺寸
2. 降低批次大小（`rec_batch_num`）
3. 关闭其他 GPU 应用
4. 切换到 CPU 模式

### Q5: 如何自定义模型？

**A**: 
目前支持 PaddleOCR 官方预训练模型。如需自定义训练，请参考 PaddleOCR 官方文档，未来版本将支持自定义模型导入。
