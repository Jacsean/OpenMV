# 相机节点使用指南

## 📋 快速开始

### 1. 添加相机节点

从左侧节点库的 **"图像相机"** 标签页中，拖拽 **"工业相机采集"** 节点到画布中央区域。

### 2. 打开预览窗口

**双击** 相机节点，立即打开实时预览窗口。

### 3. 启动相机（三种方式）

#### 方式一：自动启动（推荐）
- 双击节点后，如果相机未启动，系统会提示：
  > "相机尚未开始采集！是否要自动执行以下操作？① 初始化相机 → ② 打开相机 → ③ 开始采集"
- 点击 **"是"**，系统自动完成全部启动流程

#### 方式二：手动控制
在预览窗口顶部工具栏中，按顺序点击：
1. **🔧 初始化** - 选择Seat并初始化相机设备
2. **📡 打开** - 打开相机连接
3. **▶️ 开始采集** - 开始连续采集图像

#### 方式三：通过属性面板配置
在右侧属性面板中：
- 设置 **Seat索引**（0-4）
- 查看 **相机序列号** 和 **镜头倍率**
- 调整 **曝光时间**、**增益** 等参数

## 🎮 预览窗口功能

### 顶部工具栏

| 按钮 | 功能 | 说明 |
|------|------|------|
| 🔧 初始化 | 初始化相机设备 | 根据Seat索引加载相机配置 |
| 📡 打开 | 打开相机连接 | 建立与相机的通信 |
| ▶️ 开始采集 | 开始连续采集 | 后台线程持续抓取图像 |
| ⏸️ 停止 | 停止采集 | 暂停图像采集 |
| 💾 保存当前帧 | 保存图像到文件 | 支持JPG/PNG/BMP格式 |
| 🔄 刷新 | 手动刷新预览 | 从节点缓存获取最新帧 |

### 图像交互

- **滚轮缩放**：鼠标滚轮放大/缩小图像
- **拖拽平移**：按住空格键 + 鼠标拖拽移动视图
- **适应视图**：首次加载时自动适应窗口大小

### 底部状态栏

显示实时性能监控信息：

| 指标 | 说明 | 颜色含义 |
|------|------|----------|
| **缓冲** | 环形缓冲区使用情况 | 紫色 |
| **丢帧** | 丢帧率统计 | 🟢 <5% / 🟡 5-10% / 🔴 >10% |
| **FPS** | 实时帧率 | 蓝色 |
| **帧数** | 累计处理帧数 | 绿色 |

## 🔧 常见问题

### Q1: 双击节点没有反应？

**可能原因：**
1. 节点类型识别失败
2. `open_preview_window` 方法不存在

**解决方法：**
- 检查日志输出，确认看到 `"✅ 识别为 CameraCaptureNode"`
- 确认使用的是最新的代码版本

### Q2: 提示"相机初始化失败"？

**可能原因：**
1. Seat索引超出范围
2. 相机配置文件错误
3. 相机驱动未正确加载

**解决方法：**
- 检查 `camera_config.json` 中的Seat配置
- 确认Seat索引在有效范围内（0-4）
- 查看日志中的详细错误信息

### Q3: 预览窗口黑屏或无图像？

**可能原因：**
1. 相机未开始采集
2. 相机硬件故障
3. 连接线松动

**解决方法：**
- 确认已执行：初始化 → 打开 → 开始采集
- 检查相机硬件状态指示灯
- 重新插拔USB/网线连接

### Q4: FPS很低或丢帧率高？

**可能原因：**
1. 分辨率过高
2. 曝光时间过长
3. 系统资源不足

**解决方法：**
- 降低相机分辨率
- 减少曝光时间
- 关闭其他占用CPU的程序
- 检查环形缓冲区大小配置

## 📊 性能优化建议

### 1. 选择合适的分辨率
- 低分辨率（640x480）：适合高速检测场景
- 中分辨率（1280x720）：平衡速度与精度
- 高分辨率（1920x1080+）：适合精细测量

### 2. 调整曝光时间
- 短曝光（<10ms）：适合运动物体
- 中曝光（10-50ms）：通用场景
- 长曝光（>50ms）：低光照环境

### 3. 监控丢帧率
- **<5%**：正常范围
- **5-10%**：警告，考虑优化
- **>10%**：严重，必须优化

### 4. 使用环形缓冲区（Phase 3）
- 默认缓冲区大小：10帧
- 可根据内存调整大小
- 自动丢弃最旧帧，保证实时性

## 🚀 高级功能

### 订阅者机制

相机节点支持发布-订阅模式，其他节点可以订阅图像流：

1. **实时预览节点**：订阅并显示实时画面
2. **快速检测节点**：订阅并进行边缘检测/颜色分割
3. **视频录制节点**：订阅并保存为视频文件

### 管理订阅者

双击相机节点后，可以在预览窗口中查看和管理所有订阅者：
- 查看每个订阅者的FPS
- 查看累计接收帧数
- 取消不需要的订阅

## 📝 技术架构

### 组件关系

```
CameraCaptureNode (相机采集节点)
    ├── CameraManager (相机管理器)
    │   └── CameraDriver (相机驱动抽象)
    ├── CircularBuffer (环形缓冲区) [Phase 3]
    ├── PubSubManager (发布-订阅管理器) [Phase 3]
    └── CameraPreviewDialog (预览窗口)
        ├── 控制面板 (初始化/打开/开始/停止)
        ├── 图像显示区 (QGraphicsView)
        └── 状态栏 (FPS/丢帧率/缓冲区)
```

### 数据流

```
相机硬件 → 驱动层 → 采集线程 → 环形缓冲区 → 发布管理器 → 订阅者节点
                                    ↓
                              简单缓存 (兼容旧代码)
                                    ↓
                              预览窗口显示
```
