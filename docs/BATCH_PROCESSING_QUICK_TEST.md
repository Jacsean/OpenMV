# 批量处理功能 - 快速测试指南

## 前提条件

确保你已经在 `batch-processing-feature` 分支上：
```bash
git branch
# 应该显示: * batch-processing-feature
```

## 快速测试步骤

### 1. 启动应用程序

```bash
python src\python\main.py
```

### 2. 创建测试工作流

**步骤：**
1. 新建一个工程（Ctrl+Shift+N）
2. 从节点库拖拽以下节点到画布：
   - **加载图像**（io_camera 分类）
   - **灰度化**（preprocessing 分类，可选）
   - **Canny边缘检测**（feature_extraction 分类，可选）
   - **图像显示**（io_camera 分类）
3. 连接节点：加载图像 → [灰度化] → [Canny] → 图像显示
4. 保存工作流

### 3. 准备测试图像

在任意文件夹中准备 5-10 张测试图像：
- 格式：JPG, PNG, BMP 等
- 数量：建议先测试少量文件（5-10个）
- 位置：记住文件夹路径

### 4. 执行批量处理

**步骤：**
1. 点击工具栏上的 **"📦 批量执行"** 按钮
2. 在弹出的对话框中：
   - 点击"添加文件夹"
   - 选择包含测试图像的文件夹
   - 确认文件列表显示正确
   - 点击"确定"
3. 观察进度对话框：
   - 查看实时进度更新
   - 等待处理完成
4. 查看结果：
   - 自动弹出"批量处理结果"窗口
   - 左侧表格显示所有文件状态
   - 右侧显示详细信息

## 常见问题排查

### 问题 1: 点击"批量执行"按钮无反应

**可能原因：**
- 没有激活的工作流
- GraphEngine 未初始化

**解决方法：**
1. 确认当前有打开的工作流标签页
2. 检查工作流中是否有节点
3. 查看控制台是否有错误日志

### 问题 2: 文件添加失败

**可能原因：**
- 文件格式不支持
- 文件路径包含特殊字符
- 文件大小超限

**解决方法：**
1. 确认文件格式在白名单中（jpg/png/bmp/tif/webp）
2. 使用简单路径（避免中文、空格）
3. 检查文件大小是否超过限制（默认50MB/文件）

### 问题 3: 结果查看器不显示图像

**可能原因：**
- 工作流没有图像输出
- 节点连接不正确

**解决方法：**
1. 确认工作流中有"图像显示"节点或其他输出图像的节点
2. 检查节点连接是否正确
3. 查看JSON数据标签页，确认有图像数据输出

### 问题 4: 内存占用过高

**可能原因：**
- 文件过大或数量过多
- 缓存未及时清理

**解决方法：**
1. 减少批次中的文件数量
2. 降低图像分辨率
3. 检查控制台是否有内存清理日志
4. 重启应用释放内存
