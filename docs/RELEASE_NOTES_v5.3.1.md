# 版本发布说明 - v5.3.1

**发布日期**: 2026-05-07  
**上一版本**: v5.2.0

## 📋 版本概览

v5.3.1 是一次重要的日志系统优化更新，在 v5.2.0 基础上增强了日志输出的可追踪性，新增函数名和代码行号信息，显著提升调试效率和问题定位能力。

---

## ✨ 新增功能

### 1. 日志输出格式增强

**功能特性**:
- ✅ **函数名追踪**: 自动记录调用日志的函数名称
- ✅ **行号定位**: 自动记录调用日志的代码行号
- ✅ **多Handler支持**: 控制台、文件、UI面板均支持新格式

**输出格式变更**:

| 格式 | 示例 |
|-----|------|
| **修改前** | `[14:23:45] [INFO] [module] message` |
| **修改后** | `[14:23:45] [INFO] [module] [function][line] message` |

**实际输出示例**:
```
[12:08:47] [INFO    ] [test_module] [test_function][4] 测试信息日志
[12:08:47] [WARNING ] [plugin_loader] [load_plugins][42] 插件加载失败
[12:08:47] [ERROR   ] [node_executor] [execute][156] 节点执行超时
```

**技术实现**:
- 使用 `inspect.stack()` 遍历调用栈
- 智能跳过日志系统内部函数，准确定位业务代码位置
- 支持 `ConsoleHandler`、`FileHandler`、`QtLogHandler` 三种输出目标

## 📊 性能影响

- **性能开销**: 每次日志调用增加一次 `inspect.stack()` 调用，耗时约 0.1-0.5ms
- **建议**: DEBUG级别大量输出时可能影响性能，生产环境建议使用INFO级别
- **优化空间**: 可考虑通过配置开关控制是否启用函数追踪
