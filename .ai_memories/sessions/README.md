## 如何记录会话

### 何时记录

- ✅ 重要的架构决策讨论
- ✅ 复杂问题的排查过程
- ✅ 新功能的方案设计
- ✅ 遇到的坑及解决方案
- ❌ 日常琐碎对话无需记录

### 记录格式

```markdown
# 会话记录：YYYY-MM-DD 主题

## 背景
简要描述问题或需求的背景。

## 讨论要点
1. 关键点一
2. 关键点二
3. ...

## 决策结果
最终达成的共识或决策。

## 实施步骤
1. ✅ 已完成的任务
2. ⏳ 待完成的任务
3. ...

## 经验教训
从本次讨论中学到的经验。

## 相关文件
- `path/to/file1.py`
- `docs/xxx.md`
```

### 如何触发记录

**方式 1：主动请求**
```
用户："请记录这次关于插件迁移的讨论"
助手：生成 Markdown 文件并保存到 .ai_memories/sessions/
```

**方式 2：自动提议**
```
助手："这次讨论涉及重要的架构决策，是否需要保存到会话记录中？"
用户："是的，保存"
助手：生成文件
```

## 维护建议

1. **定期整理**：每月审查一次，删除无价值的记录
2. **添加标签**：在文件头部添加标签便于分类（如 `#架构` `#调试` `#新功能`）
3. **关联文档**：重要的会话应转化为正式文档放入 `docs/`
4. **团队共享**：通过 Git 同步，团队成员可查看历史决策
